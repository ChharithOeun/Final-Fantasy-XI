# Reactive World

> Every action has a reaction. The wiki is a guideline, not a spec.

This is the system that breaks FFXI's static-world contract. NPCs aren't furniture, quests aren't fill-in-the-blank exercises, and the economy isn't a faucet you tap. The world responds.

## Three feedback loops

### 1. NPC permadeath

If a town NPC is killed by a monster, fomor, or hostile player action:

- The NPC stays dead for **8 real-time hours** (~32 Vana'diel hours).
- The replacement is **not the same NPC**. A junior NPC takes the spot — different dialogue, possibly different inventory, lower-level quests. Comes from a pool keyed off the dead NPC's role.
- After the 8h cooldown, if the original's body is recovered (it stays in-world as a corpse), the original respawns. Otherwise the junior stays for another full cycle.

This means a fomor wave through Bastok Markets isn't just visual — it materially changes who you can talk to for a real working day.

### 2. Dynamic quest demands

Wiki says: "Bring 3 cornettes to the cooks' guild." In the static world that's the spec.

In Hardcore, the cooks' guild is run by an LLM-driven NPC who has memory of the local economy. The same quest will produce different demands depending on world state:

- **Normal**: "Bring 3 cornettes." Standard.
- **Cornette shortage** (price up 30%+): "I need 5 cornettes — and I'll pay double if you can find them. The market's been hell since the chocobo run got hit by goblins."
- **Glut** (price down 30%+): "Cornettes are everywhere right now, I don't need them. But if you bring me 2 stacks of selbina butter I'll give you the same reward."
- **Player-specific** (chharbot remembers): "You again? Fine — but this time I want you to also pick up the package my cousin in Sandy has waiting. He owes me."

The variant is generated by the NPC's LLM on quest-accept based on three inputs:

- The canonical quest spec (rewards, base demand, difficulty band)
- The current economy state (price index, zone-route safety, recent player behavior)
- The NPC's memory of this player

### 3. Economy feedback

The economy ticks every Vana'diel hour. Inputs:

- Drops players turned in to vendors (supply)
- Items players purchased (demand)
- Caravan / supply-line events (NPC trade routes between cities)
- Mob kills along trade routes (route safety affects supply)

Outputs:

- Vendor prices (real, fluctuating, displayed in-game)
- Stock levels at NPC vendors (a vendor *can* run out)
- AH price guidance (for the NPC who runs the AH listings — yes, that NPC is now an LLM agent who has opinions)
- Quest-demand shifts (feeds loop 2)

## Tooling

Most of this is **agent-based simulation** rather than ML. The classic open-source library is [`projectmesa/mesa`](https://github.com/projectmesa/mesa) — Python ABM framework — but we don't need a heavy library. The simulator is small:

```python
# in chharbot/world_sim/economy.py
class EconomyTick:
    def step(self, world_state, player_events, npc_events):
        # update price index
        # update route safety
        # update vendor stock
        # emit demand shifts to NPCs
        # persist new state
```

A single tick runs in <100ms. We tick on a 60s cadence on the chharbot server. State persists to a SQLite DB next to the existing chharbot data.

The interesting work isn't the simulator — it's the **agents reading the state**. Each NPC's prompt template gets the current economy snapshot (a 200-token JSON blob of relevant variables) injected before reflection / planning. That's what makes the cooks' guild master change his demands.

## NPC memory schema

For the dynamic quest demand to feel real, NPCs need to remember:

| Memory type | Retention | Example |
|-------------|-----------|---------|
| **Per-player interaction log** | 90 real days | "Player Tart turned in 3 cornettes on April 28. Was rude about it." |
| **World events** | 30 real days | "April 27 night: fomor wave attacked west outpost; trade road closed 6h." |
| **Personal goals** | indefinite | "I want to retire to Selbina." |
| **Relationships** | indefinite | "My cousin Mark runs the Sandy AH." |

Memory is stored per-NPC in a SQLite table with importance + recency + relevance scoring (the standard generative-agents retrieval). When the NPC needs context, top-k relevant memories get pulled into the LLM prompt.

## What this enables

A few examples of what *should* be possible after this lands:

- A fomor wave in Bastok Markets kills the cooks' guild master. For the next 8 hours the cooks' guild has a junior NPC running it — easier quests, lower rewards, and a quest dialog that mentions the master's death. After 8h the master respawns and is *angry* about it; players who report the fomor's death (because they killed it) get a permanent reputation bonus with the cooks' guild.
- Goblins start spawning more aggressively along the Bastok–Sandy chocobo route. Within hours of player casualties along that road, every cooking-related quest in either city quietly raises its rewards. Players who haven't done those quests in months notice.
- A player camps a NM for 4 hours and finally gets the drop. That NM's territory remembers — for the next 24 real hours, similar mobs in that zone fight that player slightly differently (the RL combat layer reads NM-territory tags as part of the observation).
- The Auction House NPC in each city is its own LLM agent. They form opinions: "Bastok AH's been slow this week, prices undercut." Players who chat with them get accurate market intel.

## Risk: world goes off the rails

A reactive world is also a brittle one. Mitigations:

- **Stability anchors**: critical questlines (mission progression, key items for relics) have hard caps on how much an NPC can deviate. The LLM can flavor the dialogue but can't refuse to give the cutscene.
- **Sandbox a zone first**: ship the reactive system in Bastok Markets only for the first month. Watch what happens. Adjust.
- **Cost guardrails**: every NPC LLM call is logged with token count. If a zone hits a per-day cap, the NPCs in that zone fall back to canonical quest dialog until next reset.
- **Player escape valve**: a "wiki mode" toggle in settings restores canonical quest text + skips dynamic demands. Off by default; available for hardcore players who need to grind a specific list.

## Order of build

1. **Economy simulator** — small, deterministic, easy to reason about. Persist state, expose to chharbot via MCP tools.
2. **NPC memory schema** — SQLite tables, retrieval logic. Test with one NPC before scaling.
3. **One reactive zone** — Bastok Markets. ~15 NPCs. Real economy state. Watch it run.
4. **NPC permadeath integration** — wire to the existing death pipeline.
5. **Scale to a city per month** — Sandy, Windy, Jeuno, Whitegate. Adjust as we learn.

Each step ships independently. We do not bring down the production server to land any of this.
